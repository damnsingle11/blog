JavaScript创建对象的几种模式浅析

发表于 2018-11-28 | 分类于 Javascript |

1.对象字面量

例子中创建了一个名为person的对象，并为它添加了四个属性（name，age，gender,interests）和两个方法（bio()，greeting()）。

一个如上所示的对象被称之为对象的字面量(literal)——手动的写出对象的内容来创建一个对象。但这个方法有个明显的缺点：使用同一个接口创建很多对象，会产生大量重复的代码。

2.工厂模式

该模式抽象了创建对象的具体过程，用函数来以特定接口创建对象的细节。

函数createPerson{}能够根据接受的参数构建一个包含所有必要信息的Person对象。可以无数次的调用这个函数，每次都会返回一个包含三个属性和一个方法的对象。这种模式虽然解决了创建多个相似对象的问题，却没有解决对象识别的问题。

3.构造函数模式

与工厂模式不同的是，没有显示创建对象，直接将属性和方法赋给了 this 对象，没有 return语句。需要注意的是，创建 Person的新实例时，必须使用 new 操作符。构造函数模式的问题是sayName()方法在每个实例上都会被重新创建一次。person1 和 person2都有 sayName()方法，但却不是同一个 Function实例，因此它们所共有的方法并不相等。以下代码可以证明。

4.原型模式

Javascript中创建的每个函数都有一个 prototype（原型）属性，它是一个指针，指向一个对象，包含了可以由特定类型的所有实例共享的属性和方法。

关于对象原型知识的参考：

对象原型：https://developer.mozilla.org/zh-CN/docs/Learn/JavaScript

JavaScript中创建对象的模式汇总：https://www.jb51.net/article/82644.htm

5.组合使用构造函数模式和原型模式

组合使用构造函数模式和原型模式中，构造函数用于定义实例属性，原型模型用于定义方法和共享的属性。这样每个实例都会有自己的一份实例属性的副本，同时也可以共享对方法的引用，最大限度的节省内存。

6.动态原型模式

动态原型模式将需要的所有信息都封装到构造函数中，通过 if语句判断原型中的某个属性是否存在，若不存在（在第一次调用这个构造函数的时候），执行if语句内部的原型初始化代码。
